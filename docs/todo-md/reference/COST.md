# Стоимость в токенах (Cost)

Поля стоимости фиксируют плановый и фактический расход токенов на задачу или эпик.

## Поля

- **`cost_plan`** — плановая оценка стоимости в токенах.
- **`cost_fact`** — фактическая стоимость в токенах после выполнения.

## Формат

- Значение — целое число (`integer`).
- Единица измерения — токены.
- Пустое значение допустимо, если оценка или факт ещё неизвестны.

## Правила заполнения

- `cost_plan` заполняется при планировании, если есть оценка.
- `cost_fact` заполняется после завершения работы.
- Источник факта — billing/API usage dashboard или другой журнал расхода токенов.
- В эпике `cost_plan` и `cost_fact` отражают сумму связанных задач.

## Почему токены, а не деньги

- Цена зависит от модели и провайдера.
- Токены дают стабильную базу для пересчёта.
- Перевод в деньги делается отдельно, когда известны модель и тариф.

## Пример

```yaml
cost_plan: 120000
cost_fact: 98000
```
