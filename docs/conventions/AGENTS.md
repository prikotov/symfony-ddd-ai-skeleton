# Правила для AI-агентов по работе с конвенциями (Conventions Guidelines for AI Agents)

## Назначение

Обязательные правила для AI-агентов (Codex, OpenCode, Roo, Kilo, Pi и др.) по использованию конвенций проекта, расположенных в директории `docs/conventions/`.

---

## Общие принципы

- **Язык**: русский (с английским термином в скобках при первом упоминании).
- **Точность**: без воды, только то, что поможет программисту применить подход правильно.
- **Структура**: каждый документ описывает один подход или одну сущность.
- **Код**: PHP 8.4 с использованием Symfony 8.0, строго типизированный.

---

## Как читать документы конвенций

Каждый документ в `docs/conventions/` содержит разделы:

1. **Определение** — краткое описание подхода со ссылкой на источник.
2. **Общие правила** — можно/нельзя, именование, внедрение.
3. **Зависимости** — что можно внедрять, что запрещено.
4. **Расположение** — слой и путь в иерархии папок.
5. **Как используем** — правила внедрения и ограничения.
6. **Пример** — минимальный рабочий пример на PHP.
7. **Чек-лист** — проверки для ревью.

## Правила для AI-агентов

### При написании кода

1. **Проверьте наличие конвенции** для изменяемого кода.
2. **Следуйте конвенциям** из `docs/conventions/`.
3. **Добавьте @todo** в код, если конвенция не соблюдена временно.

### При ревью кода

1. **Проверяйте соответствие** конвенциям из `docs/conventions/`.
2. **Используйте чек-листы** из документов.
3. **Аргументируйте замечания** ссылками на конкретные конвенции.

### Конвенции — внешний ресурс

Конвенции поставляются через внешний пакет и являются общими для нескольких проектов.

- **Не изменяйте** файлы в `docs/conventions/` локально — они управляются централизованно.
- Если конвенция не покрывает ваш случай — создайте issue в [prikotov/coding-standard](https://github.com/prikotov/coding-standard/issues) или обсудите с командой.
- Обновление конвенций в проекте происходит через `composer update` + повторный `coding-standard-init --force`.
