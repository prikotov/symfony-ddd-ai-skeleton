# Слой Infrastructure

Описывает технические детали: реализации репозиториев, интеграция с БД/кэшем, мапперы критериев и прочие детали, скрывающие инфраструктуру от Domain/Application.

## Общие правила

- Не содержит бизнес-логики — только техническая реализация.
- Реализует интерфейсы из Domain слоя.
- Может зависеть от сторонних библиотек и фреймворков.
- Изолирован от других модулей.

## Расположение

```
src/Module/{ModuleName}/Infrastructure/
├── Repository/
│   └── {EntityName}/
│       └── {EntityName}Repository.php
├── CriteriaMapper/
│   └── {EntityName}CriteriaMapper.php
└── Component/
    └── {ServiceName}/
        └── {ServiceName}Component.php
```

## Описание

Infrastructure слой содержит технические детали реализации: репозитории, кэш, внешние API, файловое хранилище.

## Компоненты

- [CriteriaMapper](infrastructure/criteria-mapper.md) — паттерн маппинга критериев в QueryBuilder
- [Репозитории](infrastructure/repository.md) — реализация репозиториев
- Кэш — реализация кэширования
- Внешние API — интеграция с внешними сервисами
- Файловое хранилище — работа с файлами

## Правила реализации

- Не содержит бизнес-логики
- Реализует интерфейсы из Domain слоя
- Может зависеть от сторонних библиотек
- Изолирован от других модулей

## Чек-лист для проведения ревью кода

- [ ] Infrastructure не содержит бизнес-логику.
- [ ] Репозитории реализуют доменные интерфейсы.
- [ ] CriteriaMapper изолирует логику фильтрации.
- [ ] Нет утечек Doctrine/ORM за пределы Infrastructure.
- [ ] Зависимости от сторонних библиотек обоснованы.
